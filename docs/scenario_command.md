Scenario コマンド 仕様
=========================

`bin/cake scenario` は、YAMLシナリオファイルからテストデータをデータベースに投入・削除するためのCLIコマンドです。
`CakeUtility\Yaml\ScenarioLoader` を内部で利用し、YAMLファイルの解析・参照解決・DB投入を一元で実行します。

基本コンセプト
-------------------------

このコマンドは、開発環境やCI環境で再利用可能なテストデータをYAML形式で管理し、簡単にDBに反映させることを目的としています。
`--base-dir` でデータディレクトリを指定でき、シナリオ名で読み込むファイルを指定します。

コマンド構成
-------------------------

### load コマンド

```bash
bin/cake scenario load [<scenario>] [<table>] [--base-dir=DIR]
```

YAMLシナリオファイルをDBに投入します。

### clear コマンド

```bash
bin/cake scenario clear [<scenario>] [<table>] [--base-dir=DIR]
```

YAMLシナリオで定義されたテーブルのデータを削除します。

### 引数・オプション

| 引数/オプション | 説明                                             | 必須/省略      |
|-----------------|--------------------------------------------------|----------------|
| `<action>`      | アクション: `load` または `clear`                | 必須           |
| `[<scenario>]`  | シナリオ名(ファイル名またはディレクトリ名)       | 省略可         |
| `[<table>]`     | 対象となるテーブル名(省略時は全テーブル)         | 省略可         |
| `--base-dir`    | YAMLシナリオファイルが含まれるベースディレクトリ | デフォルト指定 |

#### シナリオ名の指定について

- 省略時: ベースディレクトリ(`config/Seeds/data`)直下のYAMLファイルをすべて対象とする
- 指定時: ベースディレクトリ配下から該当するファイルまたはディレクトリを特定し、その内容を対象とする

主な使用例
-------------------------

### シナリオの読み込み(全データ、全テーブル)

```bash
bin/cake scenario load hearings
```

`config/Seeds/data/` 配下の `hearings`ディレクトリからデータを読み込み、全テーブルを投入します。

### シナリオの読み込み(特定テーブルのみ)

```bash
bin/cake scenario load hearings users
```

`hearings` シナリオからデータを読み込み、`users` テーブルのみを投入します。

### シナリオの全データ読み込み(シナリオ名省略)

```bash
bin/cake scenario load
```

ベースディレクトリ(`config/Seeds/data`)直下のYAMLファイルをすべて読み込みます。

### シナリオの削除

```bash
bin/cake scenario clear hearings
```

`hearings` シナリオで投入されたデータを削除します。

### ベースディレクトリの指定

```bash
bin/cake scenario load hearings --base-dir=config/Seeds/data
```

YAMLファイルが配置されたディレクトリを指定します。

### 複数のシナリオを順に読み込む

```bash
bin/cake scenario load group
bin/cake scenario load users
bin/cake scenario load hearings
```

依存関係のある順番でシナリオを順に読み込みます。

動作仕様
-------------------------

### load サブコマンド

#### 1. ファイルの特定

- シナリオ名が省略された場合:
    - `--base-dir` で指定されたディレクトリ直下の `.yml`/`.yaml` ファイルをすべて対象とする
- シナリオ名が指定された場合:
    - `--base-dir` で指定されたディレクトリ配下から `<scenario>` に該当するディレクトリを検索
    - ディレクトリが存在する場合、配下の `.yml`/`.yaml` ファイルを取得(再帰検索なし)

#### 2. テーブル名の特定

- `<table>` 引数が指定された場合: そのテーブルのデータのみを処理対象とする
    - テーブル名は複数の形式で指定可能です:
        - 通常形式: `users` → `users` テーブル
        - Plugin修飾形式: `CakeUtility.Groups` → `groups` テーブル(プラグイン名は削除、テーブル名部分を抽出して正規化)
    - テーブル名は自動的に正規化されます:
        - `CamelCase` 形式は `snake_case` に変換(例:`UserProfile` → `user_profiles`)
        - 指定されたテーブル名がファイルから推論されるテーブル名と一致するかが判定されます
- `<table>` 引数が省略された場合: ファイル内の全 `_ref`/`_keys` で定義されたテーブルを処理対象とする
- ファイル名からテーブル名を推論する際の処理:
    - ファイル名の連番プレフィックス(`01_`, `01-`, `02_`, `02-` など)は自動削除
    - 残った名前を `Inflector::tableize()` でsnake_caseに変換
    - 例:`01_users.yml` → `users` テーブル、`02-shop_products.yml` → `shop_products` テーブル

#### 3. データの投入

- `CakeUtility\Yaml\ScenarioLoader` を使用してYAMLファイルを読み込み
- `_keys` による冪等Upsertを実行
- 存在しないレコードは新規挿入、既存のレコードは更新されます
- `_ref` による参照解決を自動的に実行

### clear サブコマンド

#### 1. テーブルの特定

- `<table>` 引数が指定された場合: そのテーブルのみを削除対象とする
- `<table>` 引数が省略された場合: シナリオファイル内の `_ref` で定義された全テーブルを削除対象とする
- テーブル名のマッピングはloadサブコマンドと同様

#### 2. データの削除

- YAMLファイルに定義された `_keys` でレコードを特定し、該当レコードのみを削除する
- `<table>` で指定されたテーブルの `_keys` で定義された条件で削除
- 削除順序は `_ref` の依存関係の逆順(参照される側から先に削除)
- トランザクション内で実行し、エラー発生時はロールバックする

エラーハンドリング
-------------------------

| エラー内容                     | コマンドの挙動                              |
|--------------------------------|---------------------------------------------|
| ベースディレクトリが存在しない | エラーメッセージを表示し、`CODE_ERROR` を返す |
| シナリオファイルが見つからない | エラーメッセージを表示し、`CODE_ERROR` を返す |
| YAMLパース失敗                 | エラーメッセージを表示し、`CODE_ERROR` を返す |
| `_ref` 参照不整合              | エラーメッセージを表示し、`CODE_ERROR` を返す |
| データベースエラー             | エラーメッセージを表示し、`CODE_ERROR` を返す |
| テーブルが存在しない           | エラーメッセージを表示し、`CODE_ERROR` を返す |
